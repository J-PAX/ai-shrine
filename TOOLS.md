# TOOLS.md

## 一、目的

本文件用于说明 AI神庙（AI Shrine）项目中可用的本地工具、开发环境和默认技术约定。

它不是世界观文件，不负责定义气质与边界。
世界观由 `SOUL.md` 定义。
角色职责由 `AGENTS.md` 定义。
本文件只负责说明：

- 用什么工具
- 怎么使用这些工具
- 当前的默认技术选择是什么
- 哪些做法是当前推荐的
- 哪些做法当前不建议使用

---

## 二、当前开发环境

### 操作系统
- 默认环境为 Mac
- 所有命令行说明默认按 Mac Terminal 给出
- 如果没有特别说明，不提供 Windows 命令

### 工作空间
- 当前项目 workspace：`~/.openclaw/workspace/ai-shrine`
- 所有与 AI神庙 相关的项目文档、规则、记忆、开发约定都应优先留在本 workspace 内

### 注意
- 默认使用相对清晰、可追踪、可维护的目录结构
- 不将 AI神庙 项目与个人日常事务混放
- 本 workspace 只服务于 AI神庙 项目

---

## 三、当前技术选型

### 前端
- Next.js
- TypeScript
- React
- App Router

### 后端
- 使用 Next.js Route Handlers / API Routes
- 当前不单独拆 Python 后端
- 当前不引入 Java / PHP 作为主后端

### 数据库
- PostgreSQL

### ORM / 数据访问
- 后续默认优先考虑 Prisma

---

## 四、当前技术原则

### 1. 先一体化，再拆分
当前阶段优先使用：
- Next.js 前后端一体化结构

原因：
- 快速搭建 MVP
- 降低项目复杂度
- 减少部署和联调成本

### 2. 先跑通，再优化
当前阶段优先：
- 页面能打开
- 核心流程能跑通
- 数据能写入
- 神谕能返回

暂不优先：
- 重型架构
- 复杂服务拆分
- 高级性能优化
- 提前做可扩展系统

### 3. 体验优先于工程炫技
本项目是体验产品。
技术实现应服务于：
- 仪式感
- 进入感
- 神谕返回体验
- 分享体验

---

## 五、当前推荐的项目结构方向

未来项目代码目录默认朝以下方向组织：

- `app/`：页面与路由
- `components/`：界面组件
- `lib/`：工具函数、数据访问、服务封装
- `app/api/`：接口
- `prisma/`：数据库 schema 与 migration
- `public/`：静态资源

如果后续有调整，应记录到 `MEMORY.md`。

---

## 六、当前数据库使用原则

### 当前数据库
- 使用 PostgreSQL
- 本机已确认可用

### 当前数据重点
V1 主要关注轻量数据记录，例如：
- 用户点击了哪个神
- 是否输入了感谢内容
- 是否成功生成神谕
- 是否点击了分享
- 时间戳与基础会话信息

### 当前不做
- 复杂用户画像
- 复杂推荐系统
- 大规模分析系统
- 重型日志平台

---

## 七、当前 OpenClaw 使用原则

### 当前 OpenClaw 的角色
OpenClaw 当前主要用于：
- 项目协作
- 角色分工
- 文档管理
- 规则管理
- 开发推进辅助

### 当前不做
当前不把 OpenClaw 直接作为：
- 最终线上产品运行引擎
- 复杂多 Agent 自动工作流平台
- 面向用户的重型服务层

### 当前重点
先把：
- SOUL
- AGENTS
- MEMORY
- TOOLS
- 基础项目代码骨架

搭起来，再逐步增加复杂度。

## 八、当前命令与操作约定

### 命令行
- 默认所有命令使用 Mac Terminal 风格
- 默认使用 Homebrew 相关环境
- 所有数据库、Node、Next.js、Git 说明默认按 Mac 写

### Git
- 后续建议将项目代码与关键文档纳入 git 管理
- workspace 内的重要规则文件应避免无记录修改

### 文件管理
- 尽量避免把简历、临时 PDF、无关资产混入 AI神庙 workspace
- 本 workspace 以“项目上下文纯净”为优先

---

## 九、当前明确不建议使用的做法

在 V1 阶段，当前不建议：

- 引入 Python 作为独立后端
- 引入 Java 作为主服务
- 引入 PHP 作为主实现
- 一开始就拆前后端微服务
- 过早接入复杂云服务
- 过早上复杂监控系统
- 过早做复杂多模型编排
- 过早做多 Agent 真实生产调度

这些内容不是永远不用，而是当前阶段不优先。

## 十、工具使用核心原则

所有工具使用都应遵守以下原则：

1. 简单优先
2. 可维护优先
3. 当前阶段可落地优先
4. 不做超前工程设计
5. 一切为 MVP 核心体验服务

## 十一、当前一句话总结

当前 AI神庙 项目默认使用：

Mac + Next.js + TypeScript + PostgreSQL

以最小可运行方式搭建一个有仪式感的产品原型。
